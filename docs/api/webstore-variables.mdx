# Webstore Modules

<Row>
	<Col>
		Webstore variables serve as powerful tools that enable you to tailor your template, ensuring content is precisely placed within your design. Utilizing Twig as our templating engine enhances this capability by offering a flexible syntax for defining where and how content appears. For a comprehensive understanding of how to leverage Twig within your templates, including its syntax and functions, the official Twig documentation provides an invaluable resource. https://twig.symfony.com/
	</Col>
</Row>

## Product Variables 

<Row>
    <Col>
        <Fields>
            <Field name="product.name" type="string">
                Name of the product.
            </Field>
            <Field name="product.price" type="float">
                Price of the product.
            </Field>
            <Field name="product.description" type="string">
                Description of the product.
            </Field>
            <Field name="product.stock.available" type="boolean">
                Whether the product is available in stock.
            </Field>
            <Field name="product.allow_one_time_purchase" type="boolean">
                Indicates if the product can be purchased only once by a single customer.
            </Field>
            <Field name="product.image_url" type="string">
                URL of the product's image.
            </Field>
            <Field name="product.slug" type="string">
                A URL-friendly identifier (slug) for the product, used in creating web addresses for individual products.
            </Field>
            <Field name="product.pricing.active_sale" type="boolean">
                Indicates whether the product is currently on sale.
            </Field>
            <Field name="product.pricing.price_original" type="float">
                The original price of the product before any sales or discounts.
            </Field>
            <Field name="product.pricing.vat_rate" type="object">
                Details about the VAT rate applied to the product.
            </Field>
            <Field name="product.pricing.vat_rate.percentage" type="float">
                The VAT rate applied to the product price, as a percentage.
            </Field>
            <Field name="product.pricing.vat_rate.vat_abbreviation" type="string">
                Abbreviation of the VAT rate applicable to the product.
            </Field>
        </Fields>
    </Col>
</Row>


## Subscription Variables 

<Row>
    <Col>
        Files related to managing subscriptions must be placed in a `subscriptions` folder at the root directory.
    </Col>
</Row>

<Row>
    <Col>
        <Fields>
            <Field name="subscription.product_name" type="string">
                Name of the product associated with the subscription.
            </Field>
            <Field name="subscription.total_amount_str" type="string">
                The total amount of the subscription in a string format, usually including the currency symbol.
            </Field>
            <Field name="subscription.status" type="string">
                The current status of the subscription (e.g., active, pending, canceled).
            </Field>
            <Field name="subscription.interval_value" type="int">
                The numerical value of the interval between subscription charges.
            </Field>
            <Field name="subscription.interval_scale" type="string">
                The scale of the interval (e.g., day, month, year) between subscription charges.
            </Field>
            <Field name="subscription.created_at" type="string">
                The creation date of the subscription.
            </Field>
            <Field name="subscription.id" type="string">
                Unique identifier of the subscription.
            </Field>
            <Field name="subscription.active_at" type="string">
                The date when the subscription became active.
            </Field>
            <Field name="subscription.canceled_at" type="string">
                The date when the subscription was canceled, if applicable.
            </Field>
            <Field name="subscription.cancel_reason" type="string">
                The reason provided for the subscription's cancellation, if available.
            </Field>
        </Fields>
    </Col>
</Row>

<Row>
    <Col>
        <Note>
            To cancel subscriptions, use the `{{ subscription.id }}` like in the example below.
        </Note>
        ```html {{ title: 'Cancel Subscription' }}
            <form action="/subscriptions/{{ subscription.id }}/cancel" method="post">
                <button type="submit">
                    Cancel Subscription
                </button>
            </form>
        ```
    </Col>
</Row>

## Navlinks Variables

<Row>
    <Col>
        <Fields>
            <Field name="navlinks" type="array">
                An array of navigation link objects, each containing data about a specific navigation link.
            </Field>
            <Field name="navlinks[].node_id" type="string">
                Unique identifier of the navigation link.
            </Field>
            <Field name="navlinks[].name" type="string">
                Name of the navigation link.
            </Field>
            <Field name="navlinks[].tag_id" type="string">
                The unique identifier for the tag associated with the navigation link.
            </Field>
            <Field name="navlinks[].tag_query" type="array">
                An array containing query parameters used to filter products or content associated with the navigation link. Each element in the array is a string.
            </Field>
            <Field name="navlinks[].tag_slug" type="string">
                A URL-friendly slug derived from the navigation link's tag.
            </Field>
            <Field name="navlinks[].order" type="int">
                An integer representing the order in which the navigation link appears relative to other navigation links.
            </Field>
            <Field name="navlinks[].parent_node_id" type="string" nullable="true">
                The unique identifier of the parent navigation link, if any (null if the navigation link is top-level).
            </Field>
            <Field name="navlinks[].store_id" type="string">
                The unique identifier of the store to which the navigation link belongs.
            </Field>
            <Field name="navlinks[].children" type="array">
                An array of child navigation link objects, each structured the same as the parent navigation link object. This field is recursive and can contain its own `children` array.
            </Field>
        </Fields>
    </Col>
</Row>

## Carts Variables

<Row>
    <Col>
        <Fields>
            <Field name="cart" type="object">
                Represents a shopping cart, containing items chosen by a customer and other related information.
            </Field>
            <Field name="cart.store_id" type="string">
                The unique identifier of the store to which the cart belongs.
            </Field>
            <Field name="cart.customer_id" type="string">
                The unique identifier of the customer who owns the cart.
            </Field>
            <Field name="cart.currency" type="string">
                The currency in which the cart's total is denoted. This field can be an empty string if the currency is not specified.
            </Field>
            <Field name="cart.total" type="int">
                The total cost of all items in the cart.
            </Field>
            <Field name="cart.lines" type="array">
                An array of items currently in the cart.
            </Field>
            <Field name="cart.lines[].product_id" type="string">
                The unique identifier of the product.
            </Field>
            <Field name="cart.lines[].line_key" type="string">
                A unique key for the line item in the cart.
            </Field>
            <Field name="cart.lines[].name" type="string">
                The name of the product.
            </Field>
            <Field name="cart.lines[].price" type="int">
                The price of the product.
            </Field>
            <Field name="cart.lines[].quantity" type="int">
                The quantity of the product in the cart.
            </Field>
            <Field name="cart.lines[].slug" type="string">
                A URL-friendly slug for the product.
            </Field>
            <Field name="cart.lines[].image_url" type="string" nullable="true">
                The URL of the product's image. This field can be null if no image is available.
            </Field>
            <Field name="cart.lines[].selected_gameserver" type="string" nullable="true">
                The selected game server for the product, if applicable. This field can be null if no server is selected.
            </Field>
            <Field name="cart.lines[].selected_gameserver_id" type="string" nullable="true">
                The unique identifier of the selected game server, if applicable. This field can be null if no server is selected.
            </Field>
        </Fields>
    </Col>
</Row>

## Active Game Servers

<Row>
    <Col>
        <Fields>
            <Field name="available_gameservers" type="array">
                An array representing available game servers. Each element in the array is an object detailing information about a specific game server.
            </Field>
            <Field name="available_gameservers[].id" type="string">
                The unique identifier of the game server. This ID is typically used to reference the server in database queries or API calls.
            </Field>
            <Field name="available_gameservers[].name" type="string">
                The name of the game server. This name is used to represent the server to users, allowing them to select it from a list or identify it in the context of the application.
            </Field>
        </Fields>
    </Col>
</Row>

## Webstore Variables

<Row>
    <Col>
        <Fields>
            <Field name="webstore.id" type="string">
                The unique identifier of the webstore.
            </Field>
            <Field name="webstore.slug" type="string">
                A URL-friendly slug associated with the webstore.
            </Field>
            <Field name="webstore.is_active" type="boolean">
                Indicates whether the webstore is currently active (true) or not (false).
            </Field>
            <Field name="webstore.custom_domain" type="string" nullable="true">
                The custom domain set for the webstore, if any. This field can be null if no custom domain is set.
            </Field>
            <Field name="webstore.custom_domain_verified" type="boolean">
                Indicates whether the custom domain has been verified (true) or not (false).
            </Field>
            <Field name="webstore.current_template_id" type="string">
                The unique identifier of the current template being used by the webstore.
            </Field>
            <Field name="webstore.custom_templates" type="string" nullable="true">
                Information about custom templates, if any, associated with the webstore. This field can be null if there are no custom templates.
            </Field>
        </Fields>
    </Col>
</Row>

## Active Tag Variables

<Row>
    <Col>
        <Fields>

            </Field>

            </Field>
            <Field name="activeTag.slug" type="string">
                A URL-friendly slug for the active tag. Slugs are often used in URLs to represent the tag in a readable and SEO-friendly manner.
            </Field>
            <Field name="activeTag.description" type="string">
                The description of the active tag. This can include HTML content and provides more detailed information about the tag or its associated category.
            </Field>
        </Fields>
    </Col>
</Row>
